# Runtime Operations

## Purpose
Describe how to validate and operate LocalHomeApp as a local system rather than a set of isolated phase demos.

## Current runtime validation goals
- confirm required config is present
- confirm local storage/database paths are accessible
- confirm core runtime components can execute
- provide a repeatable end-to-end local demo path

## Current operational philosophy
- local-first
- privacy-first
- explicit validation over silent assumptions
- operational checks should explain failures clearly

## Planned operational tools
- health-check script
- runtime config validation script
- end-to-end pipeline demo script
